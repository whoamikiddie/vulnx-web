Overview

Vulnx is a next-generation security scanning platform that combines advanced reconnaissance techniques with powerful vulnerability assessment capabilities. Built with performance and extensibility in mind, it enables security professionals to conduct thorough security assessments efficiently.

Vulnx's modular architecture allows for efficient, parallel processing of security scanning tasks. The core components work together to provide a comprehensive security assessment platform:

  • Scanning Engine: The central component that coordinates all scanning operations. It manages parallel execution threads and optimizes resource allocation for maximum performance.
  • Workflow Manager: Defines and orchestrates scanning processes based on predefined or custom workflows. It determines the sequence and dependencies between scanning tasks.
  • Module System: Provides a plugin architecture for extending functionality. Custom scanning modules can be developed and integrated seamlessly with the core system.
  • Reporting Engine: Collects and processes scan results, generating comprehensive security reports in multiple formats with detailed vulnerability findings.
  • Configuration Manager: Handles tool settings, scan profiles, and user preferences. Enables fine-tuning of the scanning parameters for different target environments.

All components communicate through standardized APIs, allowing for seamless integration and extensibility. The modular design ensures that new features can be added without disrupting the core functionality.

Workflow Types

Vulnx supports multiple workflow types, allowing you to choose the most appropriate scanning strategy for your needs.

General

Sequential scanning for standard security assessments

vulnx scan -f general -t sample.com

Advanced

Parallel scanning with comprehensive vulnerability checks

vulnx scan -f advance -t sample.com

Archive

Scan archived content for historical vulnerabilities

vulnx scan -f archive -t example.com

Dirbscan

Directory enumeration and sensitive file discovery

vulnx scan -f dirbscan -t target.com

Usage Examples

Vulnx provides a powerful command-line interface with multiple options for different scanning scenarios. Here are some common usage examples:

# Basic scanning with a workflow
vulnx scan -f [flowName] -t [target]

# Scanning with a custom module
vulnx scan -m [modulePath] -T [targetsFile]

# Scanning with custom flow file
vulnx scan -f /path/to/flow.yaml -t [target]

# Network scanning
vulnx scan --threads-hold=30 -f cidr -t 1.2.3.4/24
# Scan with custom workspace
vulnx scan -T list_of_targets.txt -W custom_workspaces

# Debug mode scanning
vulnx scan -t target.com -w workspace_name --debug

# Aggressive tactic scanning
vulnx scan --tactic aggressive -f general -t sample.com

# Multiple target scanning
vulnx scan -f extensive -t sample.com -t another.com
# Cloud-based scanning
vulnx cloud -f [flowName] -t [target]

# Cloud module scanning
vulnx cloud -m [modulePath] -t [target]

# Multiple concurrent cloud scans
vulnx cloud -c 5 -f [flowName] -T [targetsFile]

# Cloud scanning with API token
vulnx cloud --token xxx -c 5 -f [flowName] -T [targetsFile]
